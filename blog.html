What is DOM and what is the purpose of DOM?

DOM is an acronym that stands for Document Object Model. It’s how a web browser represents a web page internally.
The DOM determines what content should be on a page and how each element of the content relates to the other elements. Let’s look at each word of the acronym.

Purpose of DOM

By manipulating the DOM, we have infinite possibilities. We can create applications that update the data of the page without needing a refresh. Also, we can create applications that are customizable by the user and then change the layout of the page without a refresh. We can drag, move, and delete elements.

How will you select HTML elements using DOM? Name the DOM methods.

DOM Selectors, as the name suggests is used to select HTML elements within a document using JavaScript. There are 5 ways in which you can select elements in a DOM using selectors.

  1. getElementsByTagName()
  2. getElementsByClassName()
  3. getElementById()
  4. querySelector()
  5. querySelectorAll()

All those methods are properties of the document object. So in order to use one, we should prefix those elements with the document object as,

  1. document.getElementByTagName()
  2. document.getElementsByClassName()
  3. document.getElementById()
  4. document.querySelector()
  5. document.querySelectorAll()

What is event bubble?

Event Bubbling is a concept in the DOM (Document Object Model). It happens when an element receives an event, and that event bubbles up (or you can say is transmitted or propagated) to its parent and ancestor elements in the DOM tree until it gets to the root element.

What is a callback function and why will you use it?

A JavaScript callback is a function which is to be executed after another function has finished execution. A more formal definition would be - Any function that is passed as an argument to another function so that it can be executed in that other function is called as a callback function.

Need of Callback functions

We need callback functions because many JavaScript actions are asynchronous, which means they don't really stop the program (or a function) from running until they're completed, as you're probably used to. Instead, it will execute in the background while the rest of the code runs.
